> **"Ne jamais construire quelque chose qui n'existe pas déjà et n'est pas déjà rentable."** — Samuel ($35K/mois)

Tous les créateurs interviewés partagent la même approche : **copier ce qui marche, puis améliorer de 1%**.

## 1. Trouver une Idée Validée

### Les 4 filtres de validation (Samuel)
1. **Je l'utiliserais moi-même** — Passion personnelle
2. **Ça marche déjà** — Preuve de marché (MRR screenshots sur Twitter)
3. **Pas de marketing massif** — Demande naturelle (pas que des pubs)
4. **Simple à maintenir** — Pas de backend complexe

### Où trouver les idées
| Source | Méthode |
|--------|---------|
| **Twitter/X** | Chercher des fondateurs qui partagent leurs MRR/Stripe screenshots |
| **TikTok** | Vérifier si des vidéos sur le problème deviennent virales |
| **Sensor Tower** | Analyser les revenus des apps concurrentes |
| **Ahrefs** | Vérifier les sources de trafic (SEO + Ads = bon signe) |
| **App Store** | Regarder les apps similaires qui cartonnent |

### Les niches les plus rentables (Stephen)
- **Santé/Bien-être** : Perte de poids, arrêt du tabac/vape, fitness
- **Argent** : Apps qui aident à gagner de l'argent (class actions, finances)
- **Productivité** : Habitudes, to-do, organisation
- **Beauté/Attractivité** : Tout ce qui améliore l'apparence

> "Les gens veulent être en meilleure santé, gagner plus d'argent, être plus attirants. Construisez des apps autour de ces désirs fondamentaux." — Stephen ($40K/mois)

## 2. Construire l'App

### Stack Technique Recommandée

| Outil | Usage |
|-------|-------|
| **React Native + Expo** | Framework cross-platform (iOS + Android) |
| **Next.js + Node.js** | Backend et site web |
| **Cursor / Claude Code** | Vibe coding avec IA |
| **Figma** | Design UI |
| **RevenueCat** | Gestion des abonnements |
| **Superwall** | A/B testing des paywalls |
| **Vercel** | Déploiement web |
| **Stripe** | Paiements |

### Approche de développement

**Connor (app $20K/mois en 14 jours) :**
1. Télécharger 20 apps concurrentes
2. Screenshot chaque écran d'onboarding
3. Les mettre côte à côte dans Figma
4. Prendre le meilleur de chaque
5. Définir ses structures de données en JSON avant de coder
6. Vibe coder avec Claude/Cursor

**Stephen (sans coder lui-même) :**
1. Brain dump toutes les idées sur Google Docs
2. Sketcher sur papier
3. Soumettre sur 99designs (50-70 designers proposent des UI)
4. Trouver un dev sur Upwork (Europe de l'Est = meilleur rapport qualité/prix)
5. Payer au projet terminé, pas à l'heure
6. Budget MVP : < $5,000 (voire < $1,000 avec templates ThemeForest)

### Règle MVP
> "Ne construis que 1-3 features principales. Ton onboarding fera 90% du travail de conversion." — Connor

## 3. L'Onboarding : La Clé de la Conversion

### Les 5 éléments d'un onboarding qui convertit (Connor)

1. **Invoquer l'émotion** — Les achats sont émotionnels, pas logiques
2. **Montrer les bénéfices** — Comment l'app va améliorer leur vie
3. **Personnalisation** — Questions qui font sentir que l'app est "pour eux"
4. **Graphiques/Charts** — Rendre l'app scientifique et légitime
5. **Social proof** — Nombre d'utilisateurs, témoignages

### L'onboarding extensif de Stephen (PuffCount)
> "Mon onboarding est long et pose beaucoup de questions. Certains disent que c'est ennuyeux, mais les données montrent que ça augmente les conversions. Les utilisateurs investissent du temps → ils sont plus engagés → ils payent."

## 4. Monétisation

### Le modèle dominant : Hard Paywall + Free Trial

| Stratégie | Résultat |
|-----------|----------|
| Soft paywall (skip possible) | Conversion faible |
| **Hard paywall (obligatoire)** | **Conversion 20-25%** |

**Format typique :**
- App gratuite à télécharger
- Onboarding extensif
- Paywall obligatoire avec free trial
- Options : Hebdo / Mensuel / Annuel
- **Pousser l'annuel** (meilleur LTV)

### A/B Testing des prix
- Utiliser **Superwall** pour tester différents prix
- Commencer à $4, monter jusqu'à $12
- Optimiser pour le **LTV** (Lifetime Value), pas le taux de conversion

## 5. Marketing : Le Vrai Levier

### La réalité brutale
> "Marketing is 95% of the success of a mobile app." — Stephen

> "Building it is just the first step. Distribution is everything." — Tous

### TikTok Organic : La Stratégie

**Étape 1 : Recherche**
1. Chercher des vidéos virales dans ta niche
2. Les sauvegarder dans un spreadsheet
3. Analyser : hook, valeur, format, durée

**Étape 2 : Créer du contenu**
- **Entertainment first, call-to-action at the end**
- Ne PAS faire de vidéo "produit" / "features"
- Copier les formats viraux et les adapter
- CTA de 2 secondes max à la fin

**Étape 3 : Scaler avec les Ads**
- Si une vidéo marche en organique → la mettre en pub
- L'algo Facebook/TikTok optimisera autour du bon créatif
- Plus le contenu est entertaining, moins le CPM est cher

### Autres canaux de croissance (Samuel)

1. **Ads** (Google, Meta, TikTok) — Validation rapide
2. **SEO** — Traffic gratuit qui compound
3. **Chaînes YouTube faceless** — UGC automatisé
4. **Programme d'affiliation** — Viralité + coût fixe

### Influenceurs (Stephen)
- Peut marcher mais coûteux et risqué
- Chercher les "diamond in the rough" créateurs
- Utiliser leur contenu pour les ads ensuite

## 6. Les Erreurs à Éviter

| Erreur | Solution |
|--------|----------|
| Passer des mois sur une idée non validée | Valider avec des screenshots MRR/Stripe |
| Construire trop de features | MVP avec 1-3 features max |
| Onboarding trop court | Onboarding extensif = plus de conversions |
| Soft paywall | Hard paywall obligatoire |
| Contenu vidéo "commercial" | Entertainment first, CTA à la fin |
| Payer les devs à l'heure | Payer au projet terminé |
| Abandonner après 1-2 mois | PuffCount : 4 ans avant le décollage |
